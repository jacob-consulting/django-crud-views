# M5 — README & Marketing Surface: Design

**Date:** 2026-07-19
**Status:** Approved design (brainstorm 2026-07-19)
**Milestone:** M5 of `superpowers/notes/2026-07-16-release-1-milestone.md`
**Ships as:** one PR, docs/CI-only, no version bump.

## Goal

The GitHub home page catches experienced Django developers and gets them from `git clone`
to a browsable example in minutes on Linux, macOS and Windows — with instructions that are
short, precise and *CI-verified working*. README, PyPI page and docs speak with one voice.

## Decisions (brainstorm outcomes)

1. **Entry path: both, tool-free first.** The README leads with a plain
   `python -m venv` + `pip install -e ".[all,examples]"` + `manage.py migrate/seed/runserver`
   path that needs nothing but Python (native Windows included), followed by a one-line
   uv+task shortcut (`task dev && task run`) for readers who have those tools.
2. **Verification: permanent CI smoke job.** A new workflow executes the README's tool-free
   path verbatim on ubuntu/macos/windows runners. Instructions stay verified forever.
3. **Hero visual: static screenshot.** One polished PNG of the example app's author list,
   generated by extending the existing `scripts/generate_screenshots.py`. No GIF.
4. **Extensions stay minimal.** No per-extension README sections; the existing one-line
   bullets and docs links stand.

## 1. README changes

Additions/edits only — the M4-aligned teaser, badges, Install, "What it is not",
Documentation and Version sections stay (Version is owned by bump-my-version).

- **Hero screenshot** directly under the badge block, above the teaser paragraph:
  `docs/getting_started/assets/readme-hero.png`, embedded via the absolute
  `https://raw.githubusercontent.com/jacob-consulting/django-crud-views/main/...` URL so
  the image renders on PyPI as well as GitHub. Alt text describes the page shown.
- **New section "Run the example project"**, placed after *Install*:
  1. Tool-free path (all platforms):
     ```
     git clone https://github.com/jacob-consulting/django-crud-views.git
     cd django-crud-views
     python -m venv .venv
     source .venv/bin/activate        # Windows: .venv\Scripts\activate
     pip install -e ".[all,examples]"
     cd examples/bootstrap5
     python manage.py migrate
     python manage.py seed
     python manage.py runserver
     ```
     then browse `http://127.0.0.1:8000/` and log in with `alice` / `alice`.
     The Windows activate variant is shown inline (comment or two-tab form — implementer's
     choice, but both variants must appear verbatim and be what CI executes).
  2. uv+task shortcut, one line each for install pointers and `task dev && task run`.
- **One-voice alignment:** compare the README's "Why developers use it" bullets with
  `docs/index.md`'s (both written from the same M4 template); align wording where they
  drifted. Small diffs only — no restructure. If a wording change is mirrored in
  `docs/index.md`, the cv-sync test and `mkdocs build --strict` must stay green.

## 2. Hero screenshot

- Extend `scripts/generate_screenshots.py` with a `readme-hero` capture: the library
  author list (table + filter form + context buttons + breadcrumb trail visible), viewport
  1280×800, logged in as `alice`. Output committed at
  `docs/getting_started/assets/readme-hero.png` like the tutorial PNGs.
- The script's existing usage/regeneration instructions cover the new capture; no separate
  tooling.

## 3. CI smoke workflow (`.github/workflows/readme-examples.yml`)

- **Matrix:** `ubuntu-latest`, `macos-latest`, `windows-latest`; single Python (3.13 via
  `actions/setup-python`).
- **Steps mirror the README tool-free path verbatim** (same commands, same order): venv,
  platform-appropriate activate, `pip install -e ".[all,examples]"`, `manage.py migrate`,
  `manage.py seed`, start `manage.py runserver` in the background, poll
  `http://127.0.0.1:8000/login/` until HTTP 200 (cross-platform Python one-liner with a
  timeout), then terminate the server. Any step failing fails the job.
- **Triggers:** `pull_request` and `push` to `main`, both path-filtered to `README.md`,
  `examples/**`, `pyproject.toml`, `src/**` and the workflow file itself.
- **Lockstep rule:** the workflow carries a header comment pointing at the README section
  it mirrors; whoever edits one updates the other. (A cv-sync-style automated gate for
  YAML↔README is deliberately out of scope.)

## 4. PyPI long description

- `pyproject.toml` already points `readme` at `README.md`; no packaging change expected.
- Verification step (implementation-time, not CI): `uv build` + `twine check dist/*`
  passes; confirm the hero image uses the raw-URL form (the PyPI-safe way to embed).

## 5. Verification & done-when

- CI smoke job green on all three OSes — this *is* the milestone's "execute the
  instructions on each platform before publishing" requirement, made durable.
- Existing suites untouched and green: package tests, examples suite (130 tests),
  `test_docs_sync.py`, `mkdocs build --strict`.
- Done when: a fresh reader on each OS can follow the README verbatim from clone to a
  browsable seeded example; README/PyPI/docs share the teaser voice; hero image renders on
  GitHub and PyPI.

## Out of scope

- Animated GIF walkthrough (rejected in brainstorm).
- Per-extension README showcase sections (rejected in brainstorm).
- Automated README↔workflow sync gate.
- Any version bump or release (README reaches PyPI with the next regular release).
